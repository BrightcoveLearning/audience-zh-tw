    使用觀眾模組追蹤檢視者

    在本主題中，您將瞭解如何使用 Audience 模組來追蹤檢視者活動。

    「觀眾」模組提供追蹤檢視活動的功能。在這種情況下檢視資料不會傳送到行銷自動化平台。其中一個常見的使用案例是內部訓練影片。受眾追蹤可用來追蹤內部員工的檢視活動。若要使用受眾追蹤，需要下列步驟：

    1. 配置受眾群體跟踪設置
    2. 配置播放器和潛在客戶表單以進行受眾群體跟踪
    3. 檢視視訊資料

    設定受眾追蹤設定

    若要設定對象追蹤設定，請依照下列步驟執行：

    1. 打開受眾模塊，然後單擊數據連接在左側導航欄中。
    2. 找到觀眾追踪部分，然後單擊節目。
    3. 根據預設，所有帳戶都會啟用對象追蹤。視需要編輯設定：
      • 連接名稱 -識別連接
      • 連接狀態 -確定“視頻雲”視頻參與度數據是否由“受眾”模塊保存。
        • 已啟用 -收集資料
        • 殘障人士 -未收集數據（連接已禁用）
      • 跟踪未知觀看者的數據 -確定是否為未知觀看者記錄了跟踪數據。為了在使用受眾群體跟踪時識別觀看者，觀眾潛在客戶表格可以使用。
        注意：透過觀眾追蹤功能，每 24 小時內最多可追蹤 1,000 位觀眾。
      • Brightcove應該保留多長時間？ -數據保留政策決定了觀看數據和銷售線索數據（如果使用受眾銷售線索表單）將在您的Video Cloud帳戶中保留多長時間。您應該根據您的隱私權政策調整保留設定，不需要在 Video Cloud 中保留任何使用者資料。

    設定玩家和潛在客戶形式，以便追蹤受眾

    觀看活動只能在具有觀眾功能的視訊播放器內觀看影片時追蹤。使用觀眾追蹤時，您必須使用已啟用受眾的播放器。您可以在影片播放期間顯示潛在客戶表單，以擷取檢視器資訊。使用觀眾追蹤時，不需要潛在客戶表單。

    若要設定受眾追蹤與啟用受眾的播放器和潛在客戶表單，請依照下列步驟執行：

    1. 按一下左側導覽中的「觀眾播放器」。
    2. 決定要使用哪個啟用受眾的播放程式。如果不存在啟用了受眾群體的播放器，請單擊創建播放器創建一個。有關創建支持受眾的播放器的更多信息，請參見創建支持觀眾的播放器。
    3. 雖然不需要潛在客戶表單，但潛在客戶表單可以用來收集觀眾的相關資訊，並將匿名觀眾變成已知的觀眾。請點擊潛在客戶表格並創建銷售線索表格。有關創建潛在客戶表單的更多信息，請參見創建受眾線索表格。
    4. 按一下觀眾播放器。
    5. 找出要使用的受眾啟用播放器。設置連接到玩家的價值觀眾追踪。
    6. 如果使用潛在客戶表格，請從潛在客戶表格下拉列表。

    檢視視訊資料

    使用觀眾追蹤時，有數個選項可供檢視視訊資料：

    • 檢視器活動 (事件) -作為「觀眾」模組的一部分，此選項可用來檢視「觀眾」模組內的視訊資料。如需資訊，請參閱檢視對象模組中的播放器事件。
    • 檢視者個人檔案 -此選項作為「對象」模組的一部分提供，可以根據「觀眾」模組內的使用者設定檔值來檢視視訊資料。有關資訊，請參閱檢視檢視器設定檔資訊。
    • 匯出資料 -作為對象模組的一部分提供，此選項可用來匯出視訊檢視和潛在客戶資料。如需資訊，請參閱從對象模組匯出受眾資料。
    • 受眾 API -受眾 API 可用於以程式設計方式擷取視訊檢視和潛在客戶資料。如需詳細資訊，請參閱概述：受眾 API 。
