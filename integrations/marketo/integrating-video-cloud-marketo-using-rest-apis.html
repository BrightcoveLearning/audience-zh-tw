    使用 REST API 整合視訊雲端與市場

    本主題說明如何設定使用 REST API 與 Marketo 搭配使用的對象模組

    觀眾模組允許視頻雲端同步從布萊特灣玩家到 Marketo 的觀看數據。一旦進入 Marketo，此資料可用於細分、行銷活動邏輯、報告、潛在客戶評分，以及個人化通訊。

    注意：有關如何使用Munchkin ID將受眾模塊配置為與Marketo一起使用的信息，請參閱使用Munchkin將Video Cloud與Marketo集成。

    與 Marketo 的受眾模組整合包含兩個主要元件：

    • 可追蹤影片觀看資料的 Brightcove 播放程式外掛程式
    • 將追蹤資料從視訊雲端傳送至 Marketo 的同步程序

    需求

    若要將受眾模組與 Marketo REST 整合搭配使用，您必須：

    • 在 Marketo 中創建一個可以訪問所有 API 權限的角色
    • 建立啟動點服務
    • 擁有專業或企業視訊雲端帳戶
    • 在開啟 Audience 模組時出現的初始啟動畫面中提交必要資訊，讓帳戶啟用 Audience 模組
    • 第一次使用模組時，按一下服務合約
    • 發布您的播放器時使用高級嵌入（頁面內）發布代碼; 標準嵌入（iframe）發布代碼將無法工作，因為插件將在 iframe 內運行，因此將無法訪問任何父頁面 URL 參數，並且可能無法獲得正確的 cookie

    建立僅限 API 的使用者和啟動點服務

    在您可以呼叫 Marketo REST API 之前，您需要建立具有存取所有 API 的新 API 角色、建立僅限 API 的使用者，然後建立唯一識別用戶端應用程式的服務。Marketo 文件中概述完成這些工作的步驟。

    設定連線

    完成 Marketo 端的設定步驟後，請依照下列步驟設定視訊雲端和 Marketo 之間的連線。

    建立視頻雲和 Marketo 之間的連接

    1. 開啟對象模組，然後按一下左側導覽中的資料連線。
    2. 按一下「新增連線」。
    3. 選取「行銷」做為整合。
      注意：可用的整合是根據您的 Video Cloud 訂閱而定。
    4. (選擇性) 輸入「連線名稱」。
    5. 按一下「連線」。
    6. 選擇標準（REST API）然後點擊保存。
      注意：有關使用Munchkin集成的信息，請參見使用Munchkin將Video Cloud與Marketo集成。

      您將需要從您的 Marketo 帳戶粘貼多個值。請注意，您必須以 Marketo 管理員身分登入，才能完成下列步驟。對於下列步驟，請按一下空白連結並提供適當的值：

    7. Marketo帳戶ID -要獲得此值，請在Marketo中單擊管理員 > 蒙奇金。
    8. Marketo REST API端點URL -要獲得此值，請在Marketo中單擊管理員 > 網頁服務。
    9. Marketo REST API身份URL -要獲得此值，請在Marketo中單擊管理員 > 網頁服務。
    10. Marketo LaunchPoint客戶端ID -要獲得此值，請在Marketo中單擊管理員 > LaunchPoint然後點擊查看詳情您之前在此設置中創建的LaunchPoint的鏈接。
    11. Marketo LaunchPoint客戶機密 -要獲得此值，請在Marketo中單擊管理員 > LaunchPoint然後點擊查看詳情您之前在此設置中創建的LaunchPoint的鏈接。
    12. 請點擊設定。

    與 Marketo 的視頻雲連接現在已經設置並可以使用。

    注意：如果沒有指派正確的權限給 Marketo 使用者，則會在 Studio 中顯示錯誤。請檢查您的權限，然後再試一次。

    設定整合

    視頻雲到 Marketo 的連接完成後，您可以配置整合。

    1. 依預設， [連線狀態] 將會是 [ 已啟用 ]。「連線狀態」會決定視訊雲端視訊參與資料是否傳送至 Marketo。
      • 啟用 -資料會收集並傳送至 Marketo
      • 停用 -未收集任何資料 (已停用整合)
    2. 根據預設，未知觀眾的追蹤資料將設為 [停用] 。啟用此選項會追蹤未知使用者的資料。資料將會根據選取的資料保留政策來保留。如果資料保留設為 [ 否保留 ]，則不會顯示此欄位。
    3. 選取資料保留原則。資料保留原則決定您所擷取的潛在客戶資訊資料是否會保留在您的 Video Cloud 帳戶和 Marketo 中，以及是否會保留多久。在 Video Cloud 中保留資料可讓您從中斷的同步處理中復原。然而，您應該根據您的隱私政策調整保留設置 -不需要在Video Cloud中保留任何用戶數據。資料將於資料保留期後刪除。

    若要移除與 Marketo 的整合，請按一下 [ 中斷連線 ]。

    建立具有受眾功能的播放器

    視訊參與資料只會在您使用支援聽眾的視訊播放器時，才會傳送至 Marketo。如需建立啟用受眾的 Brightcove 播放器的詳細資訊，請參閱建立啟用受眾的播放程式。

    識別檢視者

    具有 Marketo 整合的受眾玩家將參照在使用者瀏覽器中建立的 Munchkin Cookie (_mkto_trk)。此值會擷取在「受眾」模組的檢視資料中，並用於將檢視資料同步至 Marketo。

    用戶的電子郵件地址可以通過郵件顯示視頻的頁面上的URL參數。如需詳細資訊和範例，請參閱對象模組如何識別檢視者。

    驗證整合

    設定整合後，測試整合，以確認視訊檢視資料傳送至 Marketo。有關驗證集成的信息，請參見Marketo REST集成測試指南。
