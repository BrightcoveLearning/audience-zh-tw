    整合視訊雲端與 Oracle Eloqua

    本主題說明如何配置Audience模塊以與Oracle Eloqua一起使用。
     

    受眾模塊允許Video Cloud將Brightcove Players的觀看數據同步到Oracle Eloqua。進入Oracle Eloqua後，此數據可用於細分，活動邏輯，報告，潛在客戶評分和個性化通信。Audience模塊與Oracle Eloqua的集成包括兩個主要組件：

    • 可追蹤影片觀看資料的 Brightcove 播放程式外掛程式
    • 從Video Cloud向Oracle Eloqua發送跟踪數據的同步過程

    需求

    要將受眾模塊與Oracle Eloqua一起使用，您必須：

    • 擁有專業或企業視訊雲端帳戶
    • 在開啟 Audience 模組時出現的初始啟動畫面中提交必要資訊，讓帳戶啟用 Audience 模組
    • 第一次使用模組時，按一下服務合約
    • 發布您的播放器時使用高級嵌入（頁面內）發布代碼; 標準嵌入（iframe）發布代碼將無法工作，因為插件將在 iframe 內運行，因此將無法訪問任何父頁面 URL 參數，並且可能無法獲得正確的 cookie
    • 讓甲骨文埃洛誇 10 和憑據登錄到： https://login.eloqua.com
      注意：這些憑據將輸入到Oracle Eloqua登錄頁面中，並且不會直接提供給Brightcove。
    • 用於登錄Oracle Eloqua的證書應添加到高級用戶-營銷組並具有以下權限：
      • 使用 API
      • 安裝雲端應用程式
      • 設定行銷活動

    功能

    視頻雲-Oracle Eloqua集成的一些關鍵功能包括：

    • 潛在客戶表格 -創建Eloqua表單，這些表單會在觀眾觀看視頻時自動顯示。有關信息，請參見為Oracle Eloqua創建自定義潛在客戶表單。
    • 資料格式 -視頻雲查看數據被傳遞到Oracle Eloqua中的自定義數據對象（CDO）。複製到Oracle Eloqua的數據如下：
      • 視訊名稱
      • 視訊 ID
      • 帳號 ID (視訊雲端)
      • 網頁網址 (參照頁面的網址)
      • 玩家 ID（視頻雲端播放器）
      • % 觀看
    • 數據傳輸 -查看數據每小時同步一次，可能需要2-4個小時才能出現在Oracle Eloqua中。
    • 資料保留 -收集的數據可以選擇保留在Video Cloud中長達1個月。

    設定連線

    在配置與 Eloqua 的整合之前，我們建議您開啟新的瀏覽器視窗並手動登出您的 Eloqua 帳戶。這將確保當您配置集成時，系統會提示您登錄到您的 Eloqua 帳戶，並且您可以確認您使用的是正確的帳戶。

    請按照以下步驟在Video Cloud和Oracle Eloqua之間建立連接。

    1. 開啟對象模組，然後按一下左側導覽中的資料連線。
    2. 按一下「新增連線」。
    3. 選擇埃洛誇作為整合。
      注意：可用的數據連接取決於您的Video Cloud訂閱。
    4. (選擇性) 輸入「連線名稱」。
    5. 按一下「連線」。
    6. 按一下「連線至迴轉」。
    7. 系統會提示您登入 Eloqua。輸入您的 Eloqua 公司名稱、使用者名稱和密碼。按一下「登入」。
    8. 如果登錄成功，您將提示安裝 Eloqua 應用程序的 Bright湾視頻連接。按一下「接受並安裝」。
    9. 系統會提示您為該帳戶啟用 Eloqua。按一下「啟用」。
    10. 系統會提示您允許 Brightcove 視訊連線讓 Eloqua 存取您的 Eloqua 帳戶。按一下「登入」。
    11. 確認公司名稱和使用者名稱，然後按一下「接受」
    12. 系統將提示您設置Eloqua。請點擊設置Eloqua。

    設定整合

    一旦建立了Video Cloud與Oracle Eloqua的連接，就可以配置集成。

    1. 檢查一下將視頻觀看同步到Eloqua作為外部活動將視圖同步為外部活動。外部活動是在Eloqua中提供信息的另一個地方。發布者可以選擇使用外部活動在Eloqua中進行報告或觸發事件。有關更多信息，請參見Eloqua 文件資料。
    2. 依預設， [連線狀態] 將會是 [ 已啟用 ]。的連接狀態確定是否將Video Cloud視頻參與度數據發送到Oracle Eloqua。
      • 已啟用 -收集資料並將傳送至 Eloqua
      • 停用 -未收集任何資料 (已停用整合)
    3. 默認，跟踪未知觀看者的數據將會殘障人士。此設定會決定視訊參與資料是否儲存在 Video Cloud 中，檢視器不明時。僅將已知觀眾的數據發送到Eloqua。
      • 已啟用 -收集所有觀眾的數據並將其保存在Video Cloud中
      • 殘障人士 -未知觀眾的數據不會保存在Video Cloud中
    4. 選取資料保留原則。資料保留原則決定檢視資料和潛在客戶表單資料 (如果使用 Audience 潛在客戶表單) 將保留在您的 Video Cloud 帳戶中的時間長度。在 Video Cloud 中保留資料可讓您從中斷的同步處理中復原。此外，觀眾也會繼續重試無法同步的事件 (通常是因為觀眾在觀看影片時並不是已知的連絡人)，直到事件超過資料保留期間為止。您應該根據您的隱私權政策調整保留設定，不需要在 Video Cloud 中保留任何使用者資料。資料將於資料保留期後刪除。

    要刪除與Oracle Eloqua的集成，請單擊斷開。頁面底部將列出所有啟用了受眾群體的播放器以及相關的潛在顧客表格。

    Audience 模組支援多個行銷平台的連線。按一下 [ 新增連線 ] 以設定其他整合。

    驗證整合

    配置集成後，請測試集成以確認視頻觀看數據已發送到Eloqua。有關驗證集成的信息，請參見Eloqua集成測試指南。

    建立具有受眾功能的播放器

    僅當使用支持觀眾的視頻播放器時，視頻參與數據才會發送到Eloqua。如需建立啟用受眾的 Brightcove 播放器的詳細資訊，請參閱建立啟用受眾的播放程式。

    傳遞追蹤 ID

    受眾模塊支持將跟踪ID值傳遞給Eloqua的功能。例如，追蹤 ID 可能用來追蹤廣告活動 ID。追蹤 ID 可以附加至顯示已啟用受眾播放程式的 URL，或新增至播放程式內嵌程式碼。

    • 範例：附加至網址
      http://www.myurl.com/video.html? 按下滑鼠右鍵 = 12345
    • 範例：新增至玩家嵌入程式碼
      <視頻數據視頻代碼 = 
         資料帳戶 = "1486906377」 
         數據播放器 =「默認」 
         數據嵌入 =「默認」 
         資料應用程式識別碼 
         類 =「視頻 JS」
         資料直接追蹤器 = "12345"  
         控> < 制/視訊 >
  <指令碼的最新版 > < 本 >

    然後，受眾群體會將值作為另一個同步的字段傳遞給Eloqua。

    識別檢視者

    具有 Eloqua 整合的受眾玩家將會啟用 Eloqua 的 API 以取得訪客 ID，通常稱為 Eloqua GUID。此 ID 會擷取在「觀眾」模組的檢視資料中，並用於將檢視資料同步至 Eloqua。

    Eloqua 使用自己的一組 Cookie，當柏林灣播放器配置了 Eloqua 集成時，觀眾會在 Eloqua cookie 中捕獲 GUID 並與 Eloqua 服務器端進行通信，試圖找出用戶是誰。

    用戶的電子郵件地址也可以傳遞並用於標識查看者。如需詳細資訊和範例，請參閱對象模組如何識別檢視者。
