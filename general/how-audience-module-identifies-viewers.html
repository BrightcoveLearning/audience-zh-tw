    觀眾模組如何識別檢視者

    在本主題中，您將學習受眾模塊嘗試識別觀眾的不同方式。

    受眾群體模塊嘗試使用各種數據源來識別觀眾：

    • 受眾Cookie
    • Oracle Eloqua GUID
    • Marketo Munchkin餅乾
    • Salesforce ID URL參數
    • Salesforce ID HTML屬性
    • HubSpot UTK Cookie
    • 電子郵件參數

    受眾模塊將遵守有關以下內容的用戶瀏覽器設置：不跟踪。

    注意：使用自定義集成時，您有責任識別查看器。有關信息，請參見使用受眾模塊創建自定義集成。

    受眾Cookie

    支持受眾的播放器 如果為他們的帳戶啟用了受眾跟踪功能，並且播放器正在使用該用戶，它將設置一個第三方Cookie並將其用於將觀看數據與同一用戶關聯。如果用戶填寫了受眾線索表格，則受眾將為該查看者創建一個配置文件，並且他們的所有觀看數據都將與該配置文件相關聯。

    目標對象將設置的Cookie命名為_bc_uuid。

    Oracle Eloqua GUID

    具有 Eloqua 整合的受眾玩家將會啟用 Eloqua 的 API 以取得訪客 ID，通常稱為 Eloqua GUID。此 ID 會擷取在「觀眾」模組的檢視資料中，並用於將檢視資料同步至 Eloqua。

    Eloqua 使用自己的一組 Cookie，當柏林灣播放器配置了 Eloqua 集成時，觀眾會在 Eloqua cookie 中捕獲 GUID 並與 Eloqua 服務器端進行通信，試圖找出用戶是誰。

    Marketo Munchkin餅乾

    具有 Marketo 整合的受眾玩家將參照在使用者瀏覽器中建立的 Munchkin Cookie (_mkto_trk)。此值會擷取在「受眾」模組的檢視資料中，並用於將檢視資料同步至 Marketo。

    Salesforce ID URL參數

    對於Salesforce集成，插件可以通過檢查稱為bcsfid。這樣，Salesforce用戶可以通過在URL中提供銷售線索的Salesforce ID來發送電子郵件活動，該電子郵件活動具有為跟踪目的而唯一生成的目標頁面URL。

    Salesforce ID HTML屬性

    對於Salesforce集成，如果插件無法通過檢查bcsfid URL參數，它將搜索數據密送的屬性<video> HTML的元素嵌入視頻的發布代碼，並將其值用作潛在客戶的Salesforce ID（如果已設置）。

    HubSpot UTK Cookie

    對於 Hub S pot 整合，該外掛程式會將 HubSpot 的 JavaScript 追蹤程式碼放在使用者的瀏覽器上。此 Cookie 用於追蹤 HubSpot 中的訪客身分，並將布萊特灣查看資料與聯絡人建立關聯。如果使用者在播放程式載入時已經有 hubspotutk Cookie，觀眾將向 HubSpot 提出 API 請求，並詢問 Cookie 值是否對應於已知的連絡人，如果是這樣，觀眾會將檢視資料與該使用者建立關聯。

    電子郵件參數

    受眾模塊也可以通過電子郵件地址告知用戶。受眾群體將尋找標題為郵件。這對於具有為跟踪目的而唯一生成的著陸頁URL的電子郵件活動很有用。如果您要從營銷平台或第三方電子郵件服務（如MailChimp）向列表發送批量電子郵件，則需要在發送平台的設置中配置email參數，以動態更新用戶點擊的鏈接。然後，受眾群體將使用此電子郵件參數來查找用戶服務器端並同步觀看數據。email參數也可以嵌入播放器嵌入代碼。

    網址示例

    http://go.brightcove.com/be-video-hero-cust?bcemail=msmith@brightcove.com

    如果Audience插件沒有通過來找到電子郵件地址郵件 URL參數，然後它將檢查<video> 嵌入發布代碼的HTML元素。它將搜索數據密送電子郵件屬性設置為video元素，並將其值用作電子郵件地址（如果已設置）。

    嵌入示例

    <視頻
id =“ player”
data-account =“ 0000000001”
數據播放器 =「默認」
data-video-id =“ 0000000002”
數據密送電子郵件 =“ msmith@brightcove.com”
class =“ video-js”控件>
</ video>

    不跟踪

    啟用受眾群體的玩家必須遵守 由查看者的瀏覽器有關“不追踪（DNT）”的設置決定，如果看到“不追踪”已啟用，則不會使用或設置其正常Cookie。也可以指示播放器插件不要跟踪視頻雲客戶是否希望為其所有觀眾或逐個用戶關閉它。這是通過使用頁內嵌入代碼並設置data-do-not-track 值為1。

    <video-js data-video-id =“ 9240259371001”
data-account =“ 3491906377”
data-do-not-track =“ 1”
data-player =“ 7dccc39d-0e2d-4b27-bea9-af44b2166148”
data-embed =“默認” class =“ video-js”控件> </ video-js>
<script src =“ https://players.brightcove.net/1486906377/7dccc39d-0e2d-4b27-bea9-af44b2166148_default/index.min.js”> </ script>

    甲骨文埃洛卡

    Eloqua已經聲明他們不處理（基本上忽略）不跟踪請求，並且無論如何都會返回Eloqua GUID。當前，受眾群體沒有以任何特殊方式處理此問題，即，當受眾群體看到DNT時，不使用Eloqua JavaScript API。Eloqua還支持符合歐盟隱私要求的嚴格模式。為Eloqua帳戶啟用此功能後，除非最終用戶通過表單選擇加入，否則不會跟踪最終用戶。兩種情況下刪除的cookie都記錄在Oracle文檔中瀏覽到Eloqua跟踪頁面時會丟棄哪些Cookie。

    市場

    Marketo支持DNT，但由客戶自行為其特定帳戶打開/關閉DNT。有關更多信息，請參見Marketo文檔。編輯“不跟踪”瀏覽器支持設置。
