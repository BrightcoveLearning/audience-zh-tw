    在觀眾模組中檢視播放器事件

    在本主題中，您將學習如何在「觀眾」模組中檢視播放器賽事。

    「觀眾」模組提供追蹤檢視事件的能力。系統會根據視訊的長度動態計算檢視事件傳送至「觀眾」模組的頻率。觀眾將發送至少 4 個賽事，每四分之一。

    注意：查看事件數據可以使用匯出資料選項或使用受眾群體API。

    要查看觀看事件，請打開“受眾”模塊，然後單擊 最近所有活動在左側導航欄中。此頁面會顯示過去 24 小時內的檢視事件活動。

    的顯示互動事件如果啟用鏈接自定義事件或指標附加到查看事件。

    該頁面將顯示以下信息。

    • 時間戳記 -視頻觀看的日期和時間
    • 影片名稱和編號 -視頻名稱和視頻雲視頻ID

    • 觀看百分比 -觀看者停止觀看視頻的時間除以視頻的長度。舉例來說，如果觀眾觀看了兩分鐘影片的前 5 秒，然後跳到 55 秒，再觀看 5 秒後停止影片 (或離開頁面)，觀看的百分比會記錄為 50%。

      注意：對於實況視頻，則無法跟踪受眾群體中觀看的百分比。由於無法知道它會持續多久，因此觀眾模組只會追蹤觀看的總時間。
    • 觀看時間 -觀看的總時間
    • 外部ID -用於標識查看者的值，例如Eloqua GUID，Marketo GUID，HubSpot GUID，SSO GUID或用於標識查看者的另一個唯一值。
    • 狀態 -活動狀態
      • 未處理 -事件尚未同步到營銷自動化平台
      • 已同步 -活動已成功同步到營銷自動化平台
      • sync_disabled -受眾模塊配置為禁用同步，因此該事件被跳過
      • 失敗了 -錯誤同步；工具提示提供了有關發生了什麼錯誤的一些詳細信息
      • unknown_contact -未知用戶，因此該事件未同步
      • 不適用 - 不適用;在使用自定義，受眾跟踪或Marketo Munchkin集成時顯示

    您也可以使用頁面頂端的下拉式清單，依狀態篩選事件資料。

    檢視互動事件

    如果自定義指標或事件附加到查看事件，則顯示互動事件鏈接將被啟用。例如，如果您正在與啟用了受眾群體的播放器一起使用Gallery In-Page體驗，並且將體驗配置為鏈接，卡片或任何其他類型的互動元素，體驗會將展示和轉化事件發送到受眾模塊。這些事件會使用「受眾」自訂欄位傳遞給「聽眾」。事件數據將作為“最近所有活動。

    注意：使用時也會出現互動事件受眾群體自定義指標和事件。

    可能會傳遞下列資料：

    • 賽事1 -觀看者有機會與之互動的CTA名稱的逗號分隔列表（印象）
    • 賽事2 -觀眾以逗號分隔的CTA名稱列表做了與（互動）互動
    • 公制1 -CTA展示總數
    • 公制2 -CTA轉換的總數，即點擊鏈接，表單提交等

    要查看事件，請單擊顯示互動事件標頭中的鏈接。

    其他資料檢視選項

    觀眾資料可以透過其他方式檢視/匯出：

    • 視訊活動 -用於依影片檢視觀眾活動。如需詳細資訊，請參閱檢視視訊活動。
    • 檢視者設定檔 -用於依檢視者檢視視訊活動。有關資訊，請參閱檢視檢視器設定檔資訊。
    • 匯出資料 -用於匯出視訊檢視和潛在客戶資料。如需資訊，請參閱從對象模組匯出受眾資料。
    • 受眾 API -受眾 API 可用於以程式設計方式擷取視訊檢視和潛在客戶資料。如需詳細資訊，請參閱概述：受眾 API 。
