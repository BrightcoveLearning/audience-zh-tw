    布萊特灣觀眾常見問題

    本主題包含一些關於 Brightcove 觀眾的常見問題。

    觀眾的原因及其運作方式

    為什麼我需要在行銷自動化解決方案中追蹤影片？
    行銷自動化解決方案今天不會跟踪視頻。如果您將影片作為強大的內容投資，您應該確保追蹤影片的效率，並自動與觀看您內容的潛在客戶/使用者進行溝通。
    布萊特灣和觀眾模組是否使用任何 Cookie？
    Audience 模組會在整合到位時建立並使用第三方 Cookie 來識別使用者。如此一來，觀眾就不會被要求第二次填寫潛在客戶表單。該模塊還使用了 Marketo，HubSpot 和 Eloqua API，它們本身使用 cookie 來跟踪觀眾。使用觀眾個人檔案時，會使用「布萊特灣」曲奇檔案。
    布萊特灣觀眾是否適用於所有客戶？
    沒有觀眾可以作為視訊行銷套件 (VM) 的可購買附加元件。然而，受眾個人檔案可免費提供給每位 Brightcove 客戶。你可以把它想成「觀眾精簡版」。
    誰有資格獲得受眾個人檔案。
    自 2017 年第 4 季開始，每個虛擬機器客戶都可以免費存取受眾個人檔案。客戶將需要激活它以開始跟踪數據。企業視訊套件 (EVS) 客戶也可以使用受眾 SSO 追蹤。

    檢視者追蹤與限制

    觀眾個人檔案與功能完整的觀眾單元相比，有哪些限制？
    • 無法與任何協力廠商系統整合
    • 無 API 存取權
    • 沒有自訂潛在客戶表單 (僅限客群潛在客群表單)
    • 每 24 小時內擷取最多 1000 次檢視
    您仍然可以手動將受眾資料匯出至 CSV 或 XLS 檔案。
    當客戶連接到行銷自動化平台時，受眾有哪些限制/要求？
    每個平台的文件中概述了限制和需求：
    • 甲骨文埃洛卡
    • HubSpot（客戶端API要么REST API）
    • Marketo（蒙奇金要么REST API）
    • 銷售
    受眾 API 的輪詢頻率是否有任何限制？我知道視頻搜索結果有 1000 上限，但是該 API 端點是否有其他頻率相關的上限？
    每個 API 調用的視圖事件的最大數量實際上是 100。沒有速率限制，但如果你嘗試調用它太多，它可能會減慢一點。
    有沒有辦法比當前存在的 3 小時延遲更快獲取受眾數據？我想獲得「接近實時」的指標。
    沒有如果您想要實時指標，則需要編寫自己的插件並將數據發送到您自己的系統。
    什麼是事件視圖被發送到觀眾的觸發器？當視頻完成時，播放器窗口關閉時，還是其他東西？如果用戶從未完成視頻，並且沒有關閉他們的播放器窗口，會發生什麼情況？
    它會有所不同，但基本上，Audience 外掛程式會每 10 秒傳送一個心跳回伺服器。
    觀眾何時會考慮追蹤檢視表「完成」？是當播放器窗口關閉或者他們完成視頻？如果他們一起觀看，暫停視頻，永遠不關閉播放器窗口，會發生什麼情況？
    這是我們上次看到心跳以來 15 分鐘過去的時候，基本上缺乏活動。
    觀眾觀看資料可用後，觀眾資料是否有可能變更？未來會透過某種記錄整合來更新該事件資料嗎？
    一旦數據通過 UI 或 API 在受眾中可用，它被認為是「已解決」，它不會改變。

    資料保護與記錄保留

    受眾是否符合 GDPR 規範？

    是的，Audience 讓客戶能夠代表使用者行事，查看並清除透過 Audience 收集的任何資料。

    https://audience.support.brightcove.com/admin/managing-viewer-data.html

    資料可以在受眾中保留多久？
    MAP 整合最多可設定為 6 個月，非地圖整合可設定為 3 年。
    GUID 查詢快取多久？
    已知 30 天，未知為 7 天（僅適用於 MAP 集成，非地圖集成不需要查找）

    行銷自動化與檢視器分析

    在行銷自動化平台上追蹤影片與任何其他媒體有何不同？
    與網頁或可下載的白皮書相反，使用影片時，您可以確切判斷觀眾觀看的影片數量，因此，他們接觸到哪些內容。
    如何連接我的 Eloqua/市場/市場/HubSpot 帳戶？
    您可以直接透過觀眾模組中的視訊行銷套件登入您的 eloqua/市場/HubSpot 帳戶。從那裡，您可以使用內建 MAP 追蹤功能來建立玩家和潛在客戶表單，因此您可以立即開始追蹤影片分析。
    我可以追蹤我的 Eloqua /市場/HubSpot 頁面上的檢視次數，以及我的非 Eloqua /市場/Hubspot 登陸頁面 (網站核心/文字/圖庫等) 上的檢視次數嗎？
    可以，只要使用的玩家是「啟用觀眾」(已安裝 Audience 外掛程式)，無論玩家是否位於 MAP 登陸頁面或任何其他登陸頁面，都會追蹤所有觀看次數。
    我可以透過這項整合追蹤哪些項目？
    透過我們的整合，您可以追蹤資料庫中已知聯絡人的影片觀賞分析 (影片、影片 ID、觀看百分比、觀看時間)。
    未知的聯絡人怎麼辦？我也可以追蹤它們嗎？
    有了 Brightcove 觀眾，您可以追蹤未知的聯絡人，然後當他們提交 Eloqua/HubSpot 表單 (變為已知) 時，觀眾就能將他們的收視人數資訊與該新連絡人追溯關聯。觀眾每天都會重新提交觀看人數資訊，以便將未知聯絡人的檢視資料回 Oracle eloqua/HubSpot 執行個體。
    我的視訊檢視資料會顯示在我的地圖中？
    這取決於您使用的是哪個 MAP。請參閱以下文檔以了解甲骨文Eloqua，市場部和HubSpot。
    我的視訊觀看資料會顯示在哪裡？
    視訊檢視會以任務形式傳遞至 Salesforce，在「活動」區段中檢視者的連絡人下方。看到在Salesforce中查看視頻數據舉個例子。
    當其中一位聯絡人觀看影片時，我可以收到 Salesforce 通知嗎？
    不是直接不過，您可以執行報告，通知您連絡人的活動。
    與銷售部門追蹤與 Oracle Eloqu/ 市場的追蹤有什麼區別嗎？
    最大的區別在於，不會使用 Cookie 來追蹤使用者。因此，應傳遞唯一識別碼以識別檢視器。看到這個文件有關更多信息。此外，作為 CRM 的 Salesforce 不提供自動化後續或潛在客戶評分。不過，您可以在 Salesforce 中建立工作，然後將這些工作指派給您的銷售團隊進行實際後續作業。
    受眾有「自訂」整合選項，其他平台可以使用此功能整合哪些平台？

    觀眾可以與以下任何資料平台「整合」：

    1. 為每個連絡人記錄使用唯一識別碼
    2. 有一個 API，可以從第三方平台（如 Audience）接收其他數據

    玩家領導表單

    觀眾何時會顯示潛在客戶表單？
    預設設定只會向 MAP 資料庫中的未知使用者 (您要轉換成已知使用者的使用者) 顯示潛在客戶表單。如果您選擇永遠顯示潛在客戶表單，您可以在設定中覆寫此項目。這是測試最有用的。已知的使用者會自動追蹤，而且該資料會與 MAP 中的連絡人相關聯，而不需要顯示表單。
    我可以自訂潛在客戶表單嗎？
    您可以使用 UI 調整顯示的文字和欄位。您不能修改它超出 UI 中的可用內容（圖庫可以更新一些樣式以使表單與網站匹配）

    一般主題

    我應該使用哪個嵌入代碼？IFRAME 還是頁面？
    對象必須使用「進階」內嵌程式碼才能使用。如果使用標準 (iframe) 嵌入程式碼，則播放程式會與頁面隔離，這表示 Audience 外掛程式看不到唯一識別碼，因此，所有使用者都會被視為「未知」。
    什麼是行銷自動化？
    營銷自動化是跟踪與不同的在線營銷渠道（電子郵件，網絡，表單，內容，視頻，社交等）的領導者/用戶交互的技術。行銷自動化幫助行銷人員更有效地跨這些通道進行市場，並自動執行重複的行銷任務（例如電子郵件通訊）。營銷自動化允許營銷人員設置基於預定義的變量自動化和個性化的任務。舉例來說，如果使用者觀看特定影片的 75%，請傳送電子郵件給他們關於該特定主題的電子郵件，但如果他們只觀看 12%，請將其他內容傳送給他們。
    布萊特灣是行銷自動化公司嗎？
    不，布萊特灣提供了一個完整的視頻營銷解決方案。透過我們的解決方案，您可以在線上取得影片、組織內容、建立播放器、建立影片入口網站、追蹤分析，並與您現有的行銷自動化平台整合。Brightcove 將視訊分析整合至行銷自動化平台。
    如何檢查 Marketo 自訂活動是否正確設定？
    受眾有一個隱藏的URL，該URL為某些類型的集成提供了一些調試工具，其中Marketo REST是其中之一。 https://studio.brightcove.com/products/videocloud/audience/debug
    需要做些什麼才能配置自訂整合？
    需要做三件事：
    1. 您必須在 Audience 可擷取之網頁的 URL 或程式碼中公開唯一識別碼。
    2. 建立使用受眾 API 將資料擷取為 JSON 檔案的指令碼 (Brightcove 全球服務可以協助您解決此問題)。
    3. 將數據映射到目標數據庫中的字段（考慮電子表格合併）。
    下列支援文件涵蓋這些工作：
    • 使用受眾模塊創建自定義集成 -介紹如何提供受眾可以捕獲的唯一標識符
    • 受眾群體API -介紹如何以編程方式將所有數據作為JSON文件檢索，然後可以將其導入所選的數據處理系統
    觀眾如何將 CDO 數據推送到 Eloqua？
    我們使用自定義對象批量API。 https://docs.oracle.com/cloud/latest/marketingcs_gs/OMCAC/api-bulk-2.0-customobjects.html
    「受眾」如何推送外部資產活動？如何控制使用的廣告活動、外部資產和活動類型？
    我們使用外部活動API（ https://docs.oracle.com/cloud/latest/marketingcs_gs/OMCAC/op-api-rest-2.0-data-activity-post.html）。無法控制廣告活動名稱、資產名稱或類型。我們只需創建一個我們自己的廣告活動（名為 BrightCoveVideoplay），然後繼續創建活動，也名為 BrightCoveVideoplay。
    我可以將哪些 IP 位址列入白名單，以便觀眾可以與服務 X 交談？
    對象已建立一組應該用於 IP 白名單的 IP 位址。要查看列表，請編輯Eloqua連接，然後單擊無法在Eloqua中看到受眾群體數據？鏈接以查看IP地址列表。
